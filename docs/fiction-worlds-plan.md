## 技术方案

### 数据结构

`TRADITIONS` 数组每条增加 `category` 字段：

```js
{ id: "naruto", label: "火影忍者", glyph: "🔥", color: "#e8855a",
  sub: "Naruto", category: "fiction" }
```

现有 12 个神话传统统一标记 `category: "myth"`。世界种子 JSON 格式**不变**，仍是 10 个维度。

### 文件位置

```
public/seeds/
├── greek.json        # 已有
├── ...               # 已有 12 个
├── naruto.json       # 新增
├── hp.json
└── ...               # 新增 26 个
```

### 生成方式

在 `scripts/generate-seeds.mjs` 的 `TRADITIONS` 数组中追加新条目，重新运行：

```bash
node packages/studio/scripts/generate-seeds.mjs
```

生成 prompt 示例（架空世界需额外提示）：
```
生成世界种子：火影忍者（Naruto，岸本齐史，日本漫画，忍者世界）

注意：这是架空世界，不是真实神话传统。
请基于这个作品的内在逻辑提炼其意识形态基底，而非现实中的日本文化。
```

### UI 变化

传统选择区改为 Tab 切换：

```
[ 神话传统 · 12 ]   [ 架空世界 · 28 ]
```

架空世界内部按子类折叠/分组显示：
- 西方奇幻 (4)
- 超级英雄 (3)
- 日本动漫 (7)
- 科幻 (5)
- 中文世界 (5)
- 游戏 (4)

`NutshellUniverse.jsx` 改动点：
1. `TRADITIONS` 数组增加 `category` 字段
2. 新增 `FICTION_CATEGORIES` 分组常量
3. 选择区渲染逻辑拆分为两个 Tab
4. 每个架空世界的 `glyph` / `color` 手工指定
